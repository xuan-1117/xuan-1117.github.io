这道题目的核心在于利用纠错码理论。我们需要设计17 个问题，使得即使有 2 个回答是谎言，我们仍然能够通过纠错解码出正确得 8 位秘密。

14check-little

操作内容：

  1. 先看输入文件 output.txt

    • 打开文件，确认包含 N、c、iv、ciphertext 四项。

    • 注意 ciphertext 可能是 hex 字符串或 Python bytes 字面量。

  2. 优先尝试 gcd 分解

    • 操作：p = gcd(N, c)。

    • 思路依据：题目中 c = pow(key, e, N)。如果作者把 key 设成了 N 的某个因子（例如 key == p），那么 c 与 N 会有共同因子，gcd 能直接把 p 揪出来。

    • 若得到 1 < p < N，则成功分解；计算 q = N // p 并继续下一步。

  3. 分解成功后恢复 RSA 私钥并解密 c 得到 key

    • 计算 phi = (p - 1) * (q - 1)

    • 计算 d = inverse(e, phi)（Python: pow(e, -1, phi)）

    • 计算 recovered_key = pow(c, d, N)，这应该就是原始的 key（或等价值）。

    • 这是我最直接、最快的路径——如果能分解就基本完了。

  4. 用 recovered_key 解 AES 得到 flag

    • 根据题目代码，AES 使用的是 long_to_bytes(key)[:16] 作为 key，且是 CBC + PKCS#7。

    • 所以取 AES_key = long_to_bytes(recovered_key)[:16]，用 iv 解密 ciphertext，最后 unpad 得到 flag 字符串。

  5. 回退策略：当 gcd 失败时（我的第二条思路）

    • 因为 e = 3 很小，检查是否属于“小指数未模回绕”的情形：如果 key3 < N（即不存在模截断），那么 c == key3，可以直接对 c 做整数三次根。

    • 操作：key_candidate = integer_nth_root(c, 3)，并检查 key_candidate**3 == c。如果成立，直接按第 4 步解 AES。

    • 这是比赛中常见的低指数失误，如果作者不小心把 key 取得太小或 N 太大，会直接中招。

  6. 验证与排错（我会检查的要点）

    • 分解后检查 p * q == N。

    • 检查 recovered_key 和 p / q 的关系（题目若把 key 设为 p，应能匹配）。

    • 如果 AES 解密后的 unpad 失败，我会打印解密前的原始字节（比如前 256 字节的 hex）来判断是不是 key 取错了字节顺序或长度不足。

import argparse
import ast
import math
import logging
from typing import Optional, Tuple
from Crypto.Util.number import long_to_bytes
from Crypto.Cipher import AES
from Crypto.Util.Padding import unpad

LOG = logging.getLogger(“rsa_aes_decrypt”)

def parse_output_file(path: str) -> Tuple[int, int, bytes, bytes]:

data = {}
with open(path, “r”, encoding=”utf-8”) as f:
for lineno, line in enumerate(f, 1):
line = line.strip()
if not line or line.startswith(“#”):
continue
if “=” not in line:
LOG.debug(“skip malformed line %d: %r”, lineno, line)
continue
k, v = line.split(“=”, 1)
k = k.strip()
v = v.strip()
if k in (“N”, “c”):
data[k] = int(v)
elif k == “iv”:
# expect Python bytes literal like: b’\x01\x02…’
data[k] = ast.literal_eval(v)
elif k == “ciphertext”:
# accept hex string (with or without 0x) or Python bytes literal
if v.startswith(“b’”) or v.startswith(‘b”‘):
data[k] = ast.literal_eval(v)
else:
# remove optional 0x and whitespace
hexs = v[2:] if v.startswith(“0x”) else v
hexs = “”.join(hexs.split())
data[k] = bytes.fromhex(hexs)
else:
LOG.debug(“unknown key %s on line %d”, k, lineno)
if not all(k in data for k in (“N”, “c”, “iv”, “ciphertext”)):
missing = [k for k in (“N”,”c”,”iv”,”ciphertext”) if k not in data]
raise ValueError(f”Missing required fields in {path}: {missing}”)
return data[“N”], data[“c”], data[“iv”], data[“ciphertext”]

def integer_nth_root(x: int, n: int) -> Tuple[int, bool]:
if x < 0:
raise ValueError(“x must be non-negative”)
if x == 0:
return 0, True
low, high = 1, 1 << ((x.bit_length() // n) + 2)
while low + 1 < high:
mid = (low + high) // 2
p = mid ** n
if p == x:
return mid, True
if p < x:
low = mid
else:
high = mid
return low, low ** n == x

def recover_rsa_key_from_gcd_or_root(N: int, c: int, e: int) -> Tuple[int, Optional[int]]:
LOG.info(“Computing gcd(N, c)…”)
p = math.gcd(N, c)
if 1 < p < N:
q = N // p
LOG.info(“Found non-trivial factor p (via gcd).”)
phi = (p - 1) * (q - 1)
d = pow(e, -1, phi)
recovered = pow(c, d, N)
return recovered, d
LOG.info(“gcd did not yield a factor.”)
# fallback 1: if c is a perfect e-th power (plaintext^e), recover plaintext directly
if e and e <= 7: # small e heuristic, you can increase if desired
LOG.info(“Trying integer %d-th root of c as fallback…”, e)
r, exact = integer_nth_root(c, e)
if exact:
LOG.info(“c is exact %d-th power -> recovered plaintext directly.”, e)
return r, None
LOG.info(“Not an exact %d-th power.”, e)
# fallback 2: no recovery
raise RuntimeError(“Failed to recover key using gcd or small-exponent root fallback.”)

def aes_cbc_decrypt(aes_key: bytes, iv: bytes, ciphertext: bytes) -> bytes:
if len(iv) != AES.block_size:
raise ValueError(f”IV length must be {AES.block_size} bytes, got {len(iv)}”)
if len(aes_key) not in (16, 24, 32):
LOG.warning(“AES key length is %d - will still attempt using first 16 bytes.”, len(aes_key))
cipher = AES.new(key=aes_key, iv=iv, mode=AES.MODE_CBC)
pt_padded = cipher.decrypt(ciphertext)
return unpad(pt_padded, AES.block_size)

def short_hex_info(num: int, show_bytes: int = 16) -> str:
b = long_to_bytes(num)
if len(b) <= 2*show_bytes:
return b.hex()
return f”{b[:show_bytes].hex()}…{b[-show_bytes:].hex()} (len={len(b)} bytes)”

def main():
parser = argparse.ArgumentParser(description=”Recover AES key via RSA info then decrypt AES-CBC ciphertext.”)
parser.add_argument(“–input”, “-i”, default=”output.txt”, help=”input file containing N, c, iv, ciphertext”)
parser.add_argument(“–e”, type=int, default=3, help=”RSA public exponent (default: 3)”)
parser.add_argument(“–out”, “-o”, default=None, help=”optional file to write decrypted plaintext bytes”)
parser.add_argument(“–verbose”, “-v”, action=”store_true”)
args = parser.parse_args()

logging.basicConfig(level=logging.DEBUG if args.verbose else logging.INFO, format=”%(levelname)s: %(message)s”)

try:
N, c, iv, ciphertext = parse_output_file(args.input)
except Exception as exc:
LOG.error(“Failed to parse input: %s”, exc)
return 2

LOG.info(“Parsed values: N (bits)=%d, c (bits)=%d, iv_len=%d, ciphertext_len=%d”,
N.bit_length(), c.bit_length(), len(iv), len(ciphertext))

try:
recovered_key_int, d = recover_rsa_key_from_gcd_or_root(N, c, args.e)
except Exception as exc:
LOG.error(“RSA key recovery failed: %s”, exc)
return 3

LOG.info(“Recovered key (short hex): %s”, short_hex_info(recovered_key_int, show_bytes=16))
if d is not None:
LOG.info(“Private exponent d recovered (bit-length=%d).”, d.bit_length())
else:
LOG.info(“Recovered plaintext via small-exponent root (no RSA d).”)

# derive AES key: first 16 bytes of recovered_key_int (as original script)
recovered_bytes = long_to_bytes(recovered_key_int)
if len(recovered_bytes) < 16:
LOG.warning(“Recovered key bytes shorter than 16 (%d). Left-padding with zeros.”, len(recovered_bytes))
recovered_bytes = (b”\x00” * (16 - len(recovered_bytes))) + recovered_bytes
aes_key = recovered_bytes[:16]
LOG.info(“Using AES key (first 16 bytes) short hex: %s”, aes_key.hex())

# AES decrypt
try:
plaintext = aes_cbc_decrypt(aes_key, iv, ciphertext)
try:
text = plaintext.decode(“utf-8”)
LOG.info(“Decryption successful and decoded as UTF-8.”)
print(“\n===== Decrypted TEXT =====\n”)
print(text)
except UnicodeDecodeError:
LOG.info(“Decryption produced bytes that are not valid UTF-8. Writing raw bytes if requested.”)
if args.out:
with open(args.out, “wb”) as fw:
fw.write(plaintext)
LOG.info(“Wrote plaintext bytes to %s”, args.out)
else:
# show short hex
print(“\n===== Decrypted BYTES (hex snippet) =====\n”)
max_show = 160
snippet = plaintext[:max_show]
print(snippet.hex() + ((“… (truncated, total %d bytes)” % len(plaintext)) if len(plaintext) > max_show else “”))
# optionally save
if args.out and text is not None:
with open(args.out, “w”, encoding=”utf-8”) as fw:
fw.write(text)
LOG.info(“Wrote decoded plaintext to %s”, args.out)
except ValueError as exc:
LOG.error(“AES decryption / unpad error: %s”, exc)
LOG.debug(“Raw decrypted bytes (first 256 hex): %s”, (AES.new(key=aes_key, iv=iv, mode=AES.MODE_CBC).decrypt(ciphertext)[:256].hex()))
return 4

return 0

操作内容：

一、加密流程总览

该 flag 采用两阶段递进式加密，核心依赖 Python random模块的种子确定性 —— 相同种子可复现完全一致的随机序列，这也成为解密的关键突破口：

混淆数据gift生成：通过 3108 次循环生成随机数并计算中间结果x，拼接为gift字符串，作为种子破解的 “验证依据”；

原始 flag 混淆：将 flag 转为字符列表后，执行 2025 次shuffle洗牌操作打乱顺序，洗牌规则由同一随机数种子控制，形成最终混淆结果。

二、解密核心步骤重构

步骤 1：破解随机数种子（解密前置关键）

由于gift与种子存在强绑定关系，且 Python 随机数序列由种子唯一确定，需通过 “提取验证依据 + 暴力枚举验证” 破解种子：

提取x序列：已知gift是 3108 个x值（每个x占 2 字节）的拼接结果，因此从gift中按 2 字节为单位拆分，还原出 3108 个x值组成的验证序列；

种子枚举范围：理论上覆盖 32 位整数全范围（04294967295），但实际题目中正确种子往往取值较小，可优先快速扫描小范围（如 0100 万）提升效率；

种子验证逻辑：对每个候选种子，初始化random.Random实例，复现 3108 次循环的r1（8 位随机数）、r2（16 位随机数）生成过程，按公式x = (pow(r1, 2*i, 257) & 0xff) ^ r2计算得到候选x序列，与从gift中提取的序列完全匹配时，该候选种子即为正确种子。

步骤 2：逆向还原原始 flag（基于正确种子）

种子确定后，需通过 “复现洗牌操作 + 逆序还原” 恢复 flag 原始顺序，核心是利用洗牌算法的可逆性：

同步随机数状态：生成gift时已消耗 3108 次随机数（每次循环生成r1和r2），因此需先在正确种子初始化的random实例中，执行相同次数的随机数生成操作，确保其状态与洗牌时完全一致；

记录洗牌交换轨迹：复现 2025 次shuffle操作的完整过程 ——shuffle基于 Fisher-Yates 算法，正向逻辑为 “从后往前遍历列表，每次将第k位元素与[0, k]区间内的随机位置p交换”，记录每次交换的(k, p)位置对，形成洗牌轨迹日志；

逆序执行还原操作：从最后一次洗牌开始，按 “反向遍历洗牌次数→反向执行单次洗牌的交换步骤” 的逻辑，将每次(k, p)的交换逆转为(p, k)交换，逐步撤销 2025 次洗牌的影响，最终得到 flag 的原始字符列表，拼接后即为原始 flag。
